# 知音（职引）前端美化 Spec

## 项目信息
- 项目路径：/root/zhiyin
- 技术栈：Taro + React + TypeScript + shadcn/ui + Tailwind CSS
- 目标：美化前端页面，提升视觉体验，统一设计风格

## 现有问题分析

### 1. 风格不统一
- 首页（index）：浅色背景，卡片式布局
- 仪表盘（dashboard）：深色头部+浅色内容区
- 个人中心（profile）：浅色背景，卡片式布局
- 简历库（resume/list）：渐变色头部+浅色内容区
- **问题**：没有统一的视觉语言，每个页面风格各异

### 2. 颜色使用混乱
- 有些地方用硬编码颜色（如 `#3A4A44`）
- 有些地方用Tailwind类（如 `bg-primary`）
- 有些地方用内联style
- **问题**：维护困难，主题无法统一切换

### 3. 卡片样式不一致
- 有些卡片有 `shadow-card`
- 有些卡片只有 `shadow-sm`
- 有些卡片没有阴影
- **问题**：层次感不清晰

### 4. 间距不统一
- 有的用 `px-4`，有的用 `mx-4`
- 有的用 `mt-3`，有的用 `mt-4`
- 有的用 `mb-3`，有的用 `mb-4`
- **问题**：视觉节奏混乱

### 5. 空状态设计简陋
- 只是一个图标+一行文字
- 没有引导用户操作的按钮
- **问题**：用户体验差

### 6. 快捷入口设计单调
- 纯图标+文字，没有视觉冲击力
- 没有hover/active状态
- **问题**：不够吸引人

## 美化方案

### 设计原则
1. **统一性**：所有页面使用相同的设计语言
2. **层次感**：通过阴影、圆角、间距建立清晰的视觉层次
3. **呼吸感**：合理的留白，避免信息过载
4. **动感**：适当的动画和过渡效果
5. **现代感**：渐变色、毛玻璃、微动效

### 设计规范

#### 颜色系统
```css
/* 主色调 - 保持现有的深绿色系 */
--primary: #3A4A44;
--primary-light: #5B9A6F;
--primary-dark: #2D3A35;

/* 强调色 - 橙色系 */
--accent: #FF6B35;
--accent-light: #FFB347;

/* 背景色 */
--background: #F8FAF9;
--surface: #FFFFFF;
--surface-elevated: #FFFFFF;

/* 文字色 */
--foreground: #1A2E25;
--foreground-secondary: #6B7B74;
--foreground-muted: #9CA3AF;

/* 状态色 */
--success: #10B981;
--warning: #F59E0B;
--error: #EF4444;
--info: #3B82F6;
```

#### 阴影系统
```css
/* 卡片阴影 */
--shadow-card: 0 2px 8px rgba(0, 0, 0, 0.06);
--shadow-card-hover: 0 4px 16px rgba(0, 0, 0, 0.1);

/* 浮动元素阴影 */
--shadow-float: 0 8px 24px rgba(0, 0, 0, 0.12);

/* 下拉菜单阴影 */
--shadow-dropdown: 0 4px 12px rgba(0, 0, 0, 0.1);
```

#### 圆角系统
```css
--radius-sm: 8px;
--radius-md: 12px;
--radius-lg: 16px;
--radius-xl: 24px;
--radius-full: 9999px;
```

#### 间距系统
```css
/* 页面边距 */
--spacing-page-x: 16px;  /* px-4 */

/* 卡片内边距 */
--spacing-card: 16px;    /* p-4 */

/* 元素间距 */
--spacing-element: 12px; /* gap-3 */
--spacing-section: 20px; /* mt-5 */
```

## 动画规范

### 过渡动画
```css
/* 基础过渡 */
transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);

/* 弹性过渡 */
transition: all 0.5s cubic-bezier(0.34, 1.56, 0.64, 1);
```

### 入场动画
```css
/* 淡入上移 */
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 淡入缩放 */
@keyframes fadeInScale {
  from {
    opacity: 0;
    transform: scale(0.9);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}
```

### 微动效
```css
/* 涟漪效果 */
@keyframes ripple {
  to {
    transform: scale(4);
    opacity: 0;
  }
}

/* 脉冲效果 */
@keyframes pulse {
  0%, 100% {
    opacity: 1;
  }
  50% {
    opacity: 0.5;
  }
}
```

## 注意事项

1. **性能优先**：动画不要过度，避免影响性能
2. **兼容性**：注意Taro的兼容性，避免使用不支持的CSS属性
3. **可维护性**：使用Tailwind类，避免硬编码样式
4. **响应式**：注意不同屏幕尺寸的适配
5. **无障碍**：注意颜色对比度，确保可读性
